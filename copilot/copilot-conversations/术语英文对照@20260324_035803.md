### 1. 二进制转十进制 (Binary to Decimal)
二进制的每一位都有一个“权重”，从右向左，权重分别是 $2^0, 2^1, 2^2, 2^3, \dots$。

**方法：** 将每一位上的数字乘以它对应的权重，然后相加。

**例子：将二进制数 $1011_2$ 转换为十进制：**
- 第0位（最右边）：$1 \times 2^0 = 1 \times 1 = 1$
- 第1位：$1 \times 2^1 = 1 \times 2 = 2$
- 第2位：$0 \times 2^2 = 0 \times 4 = 0$
- 第3位（最左边）：$1 \times 2^3 = 1 \times 8 = 8$

**结果：** $8 + 0 + 2 + 1 = 11$
所以，$1011_2 = 11_{10}$。

---

### 2. 十进制转二进制 (Decimal to Binary)
最常用的方法是“除2取余法”。

**方法：** 将十进制数不断除以 2，记录下每次的余数，直到商为 0，最后将余数**逆序**排列。

**例子：将十进制数 $13$ 转换为二进制：**
1. $13 \div 2 = 6$ ... 余 **1**
2. $6 \div 2 = 3$ ... 余 **0**
3. $3 \div 2 = 1$ ... 余 **1**
4. $1 \div 2 = 0$ ... 余 **1**

**结果：** 将余数从下往上读，得到 **$1101$**。
所以，$13_{10} = 1101_2$。

**ai**: 这是一个非常好的问题，触及了**“数据在计算机中的存储方式”**与**“数据的显示方式”**之间的区别。

简单来说：**`0b100`、`0o100` 等只是你写代码时给 Python 的“输入格式”，而 `print()` 函数默认总是以“十进制”格式输出结果。**

以下是详细的原因：

### 1. 计算机内部统一存储
无论你在代码里写的是 `0b100`（二进制）、`0o100`（八进制）、`100`（十进制）还是 `0x100`（十六进制），在 Python 解释器将其读入内存时，它们**最终都会被转换成计算机底层的二进制形式存储**。

对于计算机而言，它们本质上都是同一个“数值”的不同“写法”：
- `0b100` 在内存里就是 `100`（二进制）
- `0o100` 在内存里就是 `1000000`（二进制）
- `100` 在内存里就是 `1100100`（二进制）
- `0x100` 在内存里就是 `100000000`（二进制）

### 2. `print()` 的默认行为
`print()` 函数的设计初衷是向人类展示数据。由于人类最习惯使用**十进制**，所以 Python 的 `print()` 函数在输出整数时，默认会将内存中的数值转换回十进制字符串进行显示。

这就像你用不同的语言说“苹果”：
- 你说“Apple”（英语）
- 你说“苹果”（中文）
- 你说“Pomme”（法语）

虽然表达方式不同，但你指的都是同一个水果。当你把这个水果交给别人（`print`）时，对方通常会用他最熟悉的语言（十进制）来描述它。

### 3. 如果你想看到其他进制怎么办？
如果你希望 `print()` 输出特定的进制格式，你需要使用 Python 的内置函数或格式化方法：

- **输出二进制：** 使用 `bin()` 函数
  ```python
  print(bin(0b100))  # 输出: 0b100
  ```
- **输出八进制：** 使用 `oct()` 函数
  ```python
  print(oct(0o100))  # 输出: 0o100
  ```
- **输出十六进制：** 使用 `hex()` 函数
  ```python
  print(hex(0x100))  # 输出: 0x100
  ```

### 总结
- **输入时：** 前缀（`0b`, `0o`, `0x`）是告诉 Python：“请按这个进制来解析我写的数字”。
- **输出时：** `print()` 默认只认十进制。如果你想看其他进制，必须显式地告诉 Python：“请把这个数字转换成对应的进制字符串”。

这种机制保证了代码的可读性（你可以根据需要选择最方便的进制来写数字），同时也保证了输出的统一性。

在 Python 和数学中，科学计数法遵循以下标准格式：

### 1. 数学表示法
一个数被表示为 $a \times 10^n$ 的形式，其中：
- $1 \le |a| < 10$（即 $a$ 是一个绝对值在 1 到 10 之间的数）。
- $n$ 是一个整数。

**例子：**
- $12000$ 可以写成 $1.2 \times 10^4$
- $0.00005$ 可以写成 $5 \times 10^{-5}$

---

### 2. Python 中的科学计数法
在 Python 中，我们使用字母 **`e`** 或 **`E`** 来表示“乘以 10 的多少次方”。

**语法格式：** `a e n`  等同于 $a \times 10^n$

**代码示例：**
```python
# 1.2 * 10^4
print(1.2e4)   # 输出: 12000.0

# 5 * 10^-5
print(5e-5)    # 输出: 5e-05 (即 0.00005)

# 也可以直接定义
num = 1.2e4
print(type(num)) # 输出: <class 'float'>
```

### 3. 关键点总结
- **类型是浮点数**：在 Python 中，任何使用科学计数法表示的数字，其类型（`type`）都会自动变为 `float`（浮点型）。
- **`e` 的含义**：`e` 后面跟的数字代表 10 的指数。例如 `1e3` 就是 $1 \times 10^3 = 1000$。
- **简洁性**：它非常适合处理物理学、工程学或数据科学中的极值。例如，光速约为 $3 \times 10^8$ 米/秒，在代码中写成 `3e8` 既直观又不容易出错。

以下是几个核心原因：

### 1. `e` 代表 Exponent（指数）
在数学和计算机科学中，科学计数法的标准写法是 $a \times 10^n$。这里的 $n$ 被称为**指数（Exponent）**。

当我们在代码中写 `1.2e4` 时，它的含义是：
- `1.2` 是底数（Mantissa/Coefficient）
- `e` 是 **Exponent** 的缩写，意为“以 10 为底的指数”
- `4` 是具体的指数值

所以，`e` 的作用是**连接底数和指数的桥梁**，而不是代表数字 10 本身。

### 2. 历史与键盘输入的便利性
*   **避免歧义**：如果使用 `^` 符号（在很多语言中 `^` 代表按位异或运算，而不是幂运算），或者使用 `*10**` 这种写法，代码会变得非常冗长且容易出错。
*   **历史传承**：在早期的计算机编程语言（如 Fortran、C 语言）中，为了在有限的字符空间内高效地表示浮点数，设计者选择了 `e` 作为指数的标记。这种习惯被后来的 Python、Java、JavaScript 等几乎所有主流编程语言继承了下来。

### 3. 避免与自然常数 $e$ 混淆
你可能会想到数学中的自然常数 $e \approx 2.71828$。
*   在数学公式中，如果写 $e^x$，它代表以自然常数为底的指数函数。
*   但在**编程的数字字面量**中，`e` 后面紧跟数字（如 `1e5`），编译器会明确将其识别为科学计数法。
*   如果你想在 Python 中使用数学常数 $e$，你需要通过 `import math` 然后使用 `math.e` 来调用。

### 总结
你可以这样理解：
- **`e` 不是 10**，它是“**指数标记符**”。
- 它的全称是 **Exponent**。
- 它的作用是告诉计算机：“接下来的数字是 10 的幂次方”。

**ai**: 在 Python 中，下划线（`_`）是一种**命名约定（Convention）**，用来告诉其他程序员这个变量或方法应该如何被使用。

需要明确的是：**Python 并没有像 Java 或 C++ 那样强制的“访问控制”机制**。Python 遵循“我们都是成年人”的哲学，即：它不会强行阻止你访问某些变量，但会通过命名约定来提醒你。

以下是详细解释：

### 1. 单个下划线开头：`_variable` (受保护/内部使用)
这是一种**约定**，告诉其他程序员：“这个变量是**内部使用**的，不建议在类或模块外部直接访问。”

- **含义**：它表示“受保护的”（Protected）。
- **行为**：Python 解释器**不会**对它做任何特殊处理。你依然可以在外部直接访问它，代码不会报错。
- **目的**：这是一种“君子协定”。如果你看到一个变量以 `_` 开头，你应该意识到：“我不应该直接修改它，因为它可能在未来的版本中被修改或删除。”

```python
class MyClass:
    def __init__(self):
        self._internal_data = 10  # 约定：这是内部数据

obj = MyClass()
print(obj._internal_data)  # 依然可以访问，但这是不推荐的“越界”行为
```

---

### 2. 两个下划线开头：`__variable` (私有/名称改写)
这不仅仅是约定，Python 解释器会对其进行**名称改写（Name Mangling）**，从而在一定程度上实现“私有化”。

- **含义**：它表示“私有的”（Private）。
- **行为**：Python 会自动将这个变量名修改为 `_类名__变量名`。
- **目的**：主要是为了**避免子类中的变量名与父类中的变量名发生冲突**。

```python
class MyClass:
    def __init__(self):
        self.__private_data = 20  # 私有变量

obj = MyClass()
# print(obj.__private_data)  # 报错：AttributeError，因为名字被改了

# 你可以通过这种方式访问（但强烈不建议）：
print(obj._MyClass__private_data)  # 输出: 20
```

| :--- | :--- | :--- | :--- |

| `_variable` | 受保护 (Protected) | 建议仅在内部使用 | 无特殊处理（仅是约定） |
| `__variable` | 私有 (Private) | 仅限类内部使用 | **名称改写** (Name Mangling) |

### 为什么要这样设计？
1. **封装性**：通过这些约定，开发者可以明确哪些是“接口”（可以给别人用的），哪些是“实现细节”（内部逻辑，随时可能变）。
2. **避免冲突**：`__` 的名称改写机制非常巧妙，它确保了即使在复杂的继承关系中，子类也不会意外覆盖父类的私有变量。

- 你的艺名是 `__张三`。
- 别人在外面喊 `__张三`，没人理他（报错）。
- 只有当你喊出 `_MyClass__张三` 时，你才回应他。

- 如果没有“名称改写”机制，子类的 `__data` 会直接覆盖父类的 `__data`，导致程序崩溃。
- 有了“名称改写”机制：
    - 父类的 `__data` 变成了 `_Parent__data`
    - 子类的 `__data` 变成了 `_Child__data`
